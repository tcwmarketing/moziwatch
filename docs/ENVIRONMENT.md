# Environment configuration

No secret values belong in source control. Copy `.env.example` to `.env.local` for local work and configure the equivalent encrypted variables on the deployment platform.

## Database and Supabase

| Variable                   | Development                                                                | Testing                                                                                                        | Production                                                                                                                                               | Provider and where to obtain it                                                                                                          | Can the app operate without it?                                                                                                            |
| -------------------------- | -------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| `DATABASE_URL`             | Required to use any database-backed page/API. Local Docker is the default. | The general unit suite does not need it; integration tests replace it with `TEST_DATABASE_URL`.                | Required. Use Supabase Connect → Transaction pooler for serverless/auto-scaling deployment, or Direct/Session mode for a persistent server.              | Supabase dashboard → project → **Connect** → connection string. Insert the database password and retain `sslmode=require` when supplied. | Only static pages and the fail-closed map shell work; reports, auth, campground data, forecasts, and admin do not.                         |
| `DIRECT_DATABASE_URL`      | Optional when local `DATABASE_URL` is already direct.                      | Not used for the dedicated test migration.                                                                     | Recommended and normally required operationally for migrations/backups. Use Supabase Direct connection, or Session pooler from an IPv4-only environment. | Supabase dashboard → **Connect** → Direct connection or Session pooler.                                                                  | Runtime can operate without it; `db:migrate` falls back to `DATABASE_URL`, which is not recommended when that URL is a transaction pooler. |
| `TEST_DATABASE_URL`        | Optional.                                                                  | Required for `npm run test:integration`. Must identify a separate PostGIS-enabled disposable database/project. | Never set to production.                                                                                                                                 | Create a second Supabase project/database dedicated to automated tests, then copy its connection string from **Connect**.                | Yes; only the database concurrency test is skipped/unavailable.                                                                            |
| `TEST_DATABASE_DISPOSABLE` | Leave `false` unless running the dedicated integration test.               | Must be exactly `true` for test migration/execution.                                                           | Must not be enabled in the production environment.                                                                                                       | Operator-set safety acknowledgement, not supplied by a service.                                                                          | Yes, except the destructive-capable integration test refuses to run.                                                                       |

The application uses Better Auth and Drizzle directly against PostgreSQL. It does not use Supabase Auth, Storage, Realtime, Edge Functions, or the Supabase JavaScript client. Therefore `SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_ANON_KEY`, and `SUPABASE_SERVICE_ROLE_KEY` are neither read nor required.

Enable PostGIS in Supabase under Database → Extensions → `postgis`. The migration also runs `CREATE EXTENSION IF NOT EXISTS postgis` and then creates generated geography points and GiST indexes.

## Authentication, privacy, and email

| Variable                 | Development                                                                      | Testing                                                           | Production                                                                                              | Provider and where to obtain it                                                                            | Can the app operate without it?                                                              |
| ------------------------ | -------------------------------------------------------------------------------- | ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| `BETTER_AUTH_SECRET`     | Required for auth-backed runtime; use at least 32 random bytes.                  | Unit tests do not need it; auth/database integration does.        | Required, unique, and secret.                                                                           | Generate locally with a cryptographically secure secret generator or deployment-platform secret generator. | Public static pages can render; authentication cannot operate safely.                        |
| `BETTER_AUTH_URL`        | Required; normally `http://localhost:3000`.                                      | Required only for auth integration.                               | Required; the public HTTPS application origin.                                                          | Your application/deployment URL.                                                                           | Authentication callbacks and links will be incorrect.                                        |
| `IP_HASH_SECRET`         | Required to submit reports; use a different 32+ byte value from the auth secret. | Privacy unit tests inject their own; report integration needs it. | Required and secret.                                                                                    | Generate independently with a secure secret generator.                                                     | Browsing works, but report identity hashing fails closed.                                    |
| `GOOGLE_CLIENT_ID`       | Optional unless testing Google sign-in.                                          | Required only for Google OAuth E2E tests.                         | Optional, but required to offer Google sign-in.                                                         | Google Cloud Console → APIs & Services → Credentials → OAuth 2.0 Client ID (Web application).              | Yes; email/password and other configured providers remain.                                   |
| `GOOGLE_CLIENT_SECRET`   | Same as client ID.                                                               | Same as client ID.                                                | Secret counterpart required with Google sign-in.                                                        | Same Google OAuth credential page.                                                                         | Yes, if Google sign-in is not offered.                                                       |
| `FACEBOOK_CLIENT_ID`     | Optional unless testing Facebook sign-in.                                        | Required only for Facebook OAuth E2E tests.                       | Optional, but required to offer Facebook sign-in.                                                       | Meta for Developers → app → Facebook Login settings.                                                       | Yes; email/password and other configured providers remain.                                   |
| `FACEBOOK_CLIENT_SECRET` | Same as client ID.                                                               | Same as client ID.                                                | Secret counterpart required with Facebook sign-in.                                                      | Meta for Developers → app settings → Basic.                                                                | Yes, if Facebook sign-in is not offered.                                                     |
| `RESEND_API_KEY`         | Optional; when absent, verification/reset links are logged locally.              | Required only for live email E2E tests.                           | Required for email verification and password resets.                                                    | Resend dashboard → API Keys → Create API Key after verifying a sending domain.                             | The public map can operate, but production email authentication flows fail closed.           |
| `RESEND_FROM`            | Optional locally; defaults to Resend's sandbox sender.                           | Optional unless asserting delivery.                               | Required operationally and must use a verified Resend domain, e.g. `Camp Signal <account@example.com>`. | Choose an address on the domain verified in Resend → Domains.                                              | A sandbox sender may work only for restricted testing; real production delivery requires it. |

The implemented authentication library is **Better Auth 1.6.x** with the Drizzle PostgreSQL adapter and Argon2id password hashing. Supabase is the managed PostgreSQL host only; changing to Supabase Auth is not required or planned. Email is sent through the Resend HTTP API and does not use SMTP credentials.

## Map and public application

| ----------------------------------- | --------------------------------------------------------------- | ---------------------------- | ------------------------------------------------------------------------------------ | ----------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `NEXT_PUBLIC_APP_NAME`              | Optional; defaults to `Camp Signal`.                            | Optional.                    | Optional branding value.                                                             | Product owner.                                        | Yes.                                                                                     |
| `NEXT_PUBLIC_APP_URL`               | Recommended for metadata/canonical URLs; defaults to localhost. | Optional.                    | Required for correct production metadata and links.                                  | Deployment public URL.                                | Pages render, but generated absolute URLs may be wrong.                                  |

## Forecast and scheduling

| Variable                | Development                                                        | Testing                             | Production                                                                                            | Provider and where to obtain it                                     | Can the app operate without it?                                                     |
| ----------------------- | ------------------------------------------------------------------ | ----------------------------------- | ----------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `OPEN_METEO_BASE_URL`   | Optional; defaults to the public non-commercial forecast endpoint. | Optional; provider tests mock data. | Required operationally if a paid/commercial or self-hosted endpoint is appropriate.                   | Open-Meteo customer dashboard/contract or self-hosted installation. | The default endpoint is used, but its licence/usage tier must match the deployment. |
| `OPEN_METEO_API_KEY`    | Optional for the public development endpoint.                      | Optional.                           | Required only when the chosen Open-Meteo endpoint/plan requires it.                                   | Open-Meteo commercial customer account.                             | Yes when the selected endpoint accepts unauthenticated requests.                    |
| `FORECAST_MODEL_PATH`   | Optional; defaults to `./config/models/current.json`.              | Optional.                           | Optional unless the artifact is mounted elsewhere.                                                    | Repository/deployment artifact path.                                | Yes; the checked-in `mosquito-weather-beta-v1` artifact is used.                    |
| `FORECAST_CRON`         | Optional; defaults to `15 5 * * *` UTC.                            | Optional.                           | Required only by the long-running scheduler process; platform cron can invoke `forecast:run` instead. | Operator-defined cron expression.                                   | The web app operates, but no automatic daily forecast is produced.                  |
| `FORECAST_GRID_DEGREES` | Optional; defaults to `2.5` and accepts 0.25–5.                    | Optional.                           | Optional performance/resolution tuning.                                                               | Operator/model configuration.                                       | Yes; the default grid is used.                                                      |

The checked-in beta artifact is a weather-only suitability index and has `usesUserReports=false`. It is not represented as trained. `model:train` remains the replacement workflow for a later reviewed logistic model.
